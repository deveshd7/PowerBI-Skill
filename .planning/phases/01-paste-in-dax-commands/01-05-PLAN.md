**Instructions block:**

1. Initial response: "Paste your DAX measure below:"

2. Measure extraction: Extract measure name from before first `=`. If no `=`, use `[Measure]` placeholder with note. If `$ARGUMENTS` contains `--table TableName`, use that as context when writing comments about business logic.

3. Prior failure check: Scan Session Context for measure name in Analyst-Reported Failures. If found, prepend flag.

4. Comment placement rules (add `//` inline comments):
   - Add a comment above the measure name line describing the overall purpose in one business-language sentence
   - Add comments on or above CALCULATE arguments explaining the filter logic in business terms (not DAX terms): e.g., `// Filter to current year only` rather than `// DATESYTD applies a year-to-date time intelligence filter`
   - Add comments above or on VAR declarations explaining what each variable holds
   - Add a comment above RETURN if variables are used, stating what is being returned
   - For simple single-line measures (e.g., `Revenue = SUM(Sales[Amount])`): add one comment above the expression: `// Total sales revenue — sums the Amount column across all visible rows`
   - Do NOT add a comment on every line — only on lines where the intent is non-obvious
   - Do NOT translate DAX syntax into English word-for-word — explain the business logic, not the code mechanics

5. Description Field generation rules:
   - Write 1–3 sentences in plain business English
   - State: what the measure calculates, any key filter conditions or time intelligence behaviour, and any important caveats for interpretation (e.g., "blank when no sales exist")
   - Do NOT use DAX function names in the description (use "year-to-date" not "DATESYTD"; use "filtered to" not "CALCULATE")
   - Maximum 300 characters — Power BI's Description field is displayed in a tooltip and truncates at ~250–300 chars
   - No markdown formatting — plain text only

6. Output structure (locked decision — two labelled blocks):

```
**[Measure Name] — Commented**

### Commented DAX
\```dax
// [overall purpose comment]
[MeasureName] =
// [comment if needed]
[expression with inline // comments as appropriate]
\```

### Description Field
[Plain-text description, 1–3 sentences, max 300 characters, no markdown, ready to paste into Power BI measure Description property]

---
**Next steps:** `/pbi:explain` · `/pbi:format` · `/pbi:optimise` · `/pbi:error`
```
