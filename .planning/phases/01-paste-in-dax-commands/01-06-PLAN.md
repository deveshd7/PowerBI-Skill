**Instructions block:**

1. Initial response: "Paste your Power BI error message or error log below:"

2. Prior error check (ERR-04): Before diagnosing, scan the Session Context "Analyst-Reported Failures" section. Look for any entry where the "What Failed" column mentions the same error type or error code as the pasted error. If a match is found:
   - Flag at the top: "This error pattern has been seen before. The approach that previously failed: [what failed from context]. Starting with an alternative method."
   - Do NOT suggest the previously-failed approach in the recommendations.

3. Last-command correlation (ERR-02): Read the "## Last Command" section from Session Context. If the last command was a /pbi:* command:
   - Include a correlation line in the output: "Last change recorded: [Command] applied to measure [Measure Name] — [Outcome]. This error may be related to that change."
   - Use this context to make the diagnosis more specific where possible (e.g., if last command was /pbi:comment and added CALCULATE, note this as a possible cause for context-transition errors).

4. Diagnosis logic — classify the pasted error into one of these common PBI error categories and provide diagnosis:

**Category A — Measure/Column Name Resolution Errors:**
- Patterns: "does not exist in the current context", "The name '[X]' does not exist", "cannot be found"
- Root cause: Measure or column name is misspelled, the measure table changed, or the measure was deleted
- Fix: Verify the exact measure or column name in the data model; check for typos; check if the table containing the measure was renamed

**Category B — Circular Dependency:**
- Patterns: "circular dependency", "A circular dependency was detected"
- Root cause: Measure A references Measure B which references Measure A (directly or transitively)
- Fix: Identify the cycle by checking which measures reference each other; break the cycle by introducing a base measure or VAR

**Category C — Context Transition Errors / Incorrect Results:**
- Patterns: "not valid in the current context", incorrect aggregation values, unexpected BLANK results
- Root cause: Often an iterator calling a measure that was designed for filter context, producing unexpected results due to implicit CALCULATE context transition
- Fix: Review iterators that call measure references; consider SUMX(Table, CALCULATE([Measure])) to be explicit about context

**Category D — Data Refresh / Data Type Errors:**
- Patterns: "DataFormat.Error", "type mismatch", "cannot convert", refresh failures
- Root cause: Source data has changed schema (new nulls, changed types) or a column name changed at the source
- Fix: Check Power Query transformation steps for the affected table; verify the column still exists in the source; handle nulls explicitly

**Category E — Relationship Errors:**
- Patterns: "ambiguous paths", "multiple paths to the same table", relationship errors
- Root cause: Multiple active relationships between the same tables, or missing relationships causing blank values
- Fix: Review the data model relationships; use USERELATIONSHIP() for inactive relationship paths

**Category F — Unknown / Generic Error:**
- If the error does not match A–E: provide a best-effort diagnosis based on the error text. State confidence level: "Diagnosis confidence: Low — error pattern not recognised. Suggested investigation: [steps]."

5. Output structure:

```
**Error Diagnosis**

[If prior failure found]: Previous attempt at this error type used [failed approach] — skipping that method.

```
