Add the file-mode instruction block to pbi-load/SKILL.md (appended after the no-project path instructions from Task 1).

**Step 1 — Output the file mode header immediately:**

```
File mode — PBIP project detected ([FORMAT]) | Loading model context...
```

Where [FORMAT] is "TMDL" when PBIP_FORMAT=tmdl, "TMSL (model.bim)" when PBIP_FORMAT=tmsl.

**Step 2 — Read model files and extract structure:**

For TMDL (PBIP_FORMAT=tmdl):
- The PBIP File Index injection (from Task 1) already lists the .tmdl file paths.
- Use the Read tool to read each .tmdl file from the list.
- From each file, extract:
  - Table name: from the `table TableName` declaration at the top of the file
  - Measure names: lines matching the pattern `measure 'Name'` or `measure Name` (single-word). Use the text after `measure ` up to ` =` or end of line, stripping single quotes if present.
  - Column names: lines matching the pattern `column 'Name'` or `column Name`. Same extraction rule.
- Also read `.SemanticModel/definition/relationships.tmdl` if it exists (use Read tool). Extract from/to table and column pairs from lines containing `fromTable:`, `fromColumn:`, `toTable:`, `toColumn:`.

For TMSL (PBIP_FORMAT=tmsl):
- Read `.SemanticModel/model.bim` with the Read tool.
- Navigate the JSON structure: `model.tables[]` → for each table extract `name`, `measures[].name`, `columns[].name`.
- Extract relationships from `model.relationships[]` → `fromTable`, `fromColumn`, `toTable`, `toColumn`.

**Disambiguation rule (TMDL only):** If the same measure name appears in multiple table files, log it as: `[MeasureName] (found in: Table1, Table2)` in the Measures column of the summary table. Do not fail — report all locations.

**Step 3 — Format the Model Context section:**

Build the following markdown block:

```markdown
## Model Context
**Loaded:** [current UTC time in ISO 8601 format]
**Format:** [TMDL or TMSL (model.bim)]
**Project:** .SemanticModel

| Table | Measures | Columns |
|-------|----------|---------|
| [TableName] | [measure1, measure2] | [col1, col2] |
| [TableName] | (none) | [col1, col2] |

**Relationships summary:** [FromTable][FromColumn] → [ToTable][ToColumn] (many-to-one) · [additional relationships separated by ·]
```

If no relationships file/data is found: omit the Relationships summary line entirely.
If a table has no measures: write `(none)` in the Measures column.

**Step 4 — Write .pbi-context.md (Read-then-Write):**

1. Read `.pbi-context.md` using the Read tool.
2. Check if `## Model Context` section already exists:
   - If yes: replace everything from `## Model Context` through the end of that section (up to the next `##` heading or end of file) with the new Model Context block.
   - If no: append the new Model Context block after the last existing section.
3. Write the entire updated file back using the Write tool.
4. CRITICAL: Do not remove or modify `## Last Command`, `## Command History`, or `## Analyst-Reported Failures` sections. Only the `## Model Context` section is added or updated.

**Step 5 — Output the summary table to the analyst:**

After writing .pbi-context.md, output:

```
File mode — PBIP project detected ([FORMAT]) | Context loaded.

| Table | Measures | Columns |
|-------|----------|---------|
[same table as written to .pbi-context.md]

**Relationships summary:** [same as written]

Context loaded — all DAX commands will now use model-aware analysis.
```

If PBIP_FORMAT=tmsl, include: "Format: TMSL (model.bim)" in the header line.
If PBIP_FORMAT=tmdl, include: "Format: TMDL" in the header line.

**Step 6 — Update Last Command in .pbi-context.md:**

The write in Step 4 must also update the `## Last Command` section:
- Command: `/pbi:load`
- Timestamp: current UTC time
- Measure: `[N tables loaded]` where N is the count of tables found
- Outcome: `Model context loaded ([FORMAT])`

And append a row to `## Command History`:
- Same values as Last Command
- Keep history to 20 rows max (remove oldest if exceeded)

Note: Steps 4 and 6 can be combined into a single Read-then-Write pass — read once, build the complete updated file content (updating Model Context section + Last Command + Command History), write once.
