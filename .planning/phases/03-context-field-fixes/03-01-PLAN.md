Read `.claude/skills/pbi/commands/optimise.md` and locate Step 9: Update .pbi-context.md (around line 249).

The current `## Last Command` update instruction (around lines 254-259) is:
```
2. Update the **Last Command** section:
   - Command: `/pbi optimise`
   - Measure: [measure name]
   - Rules applied: [list of rule numbers applied, e.g. Rule 1, Rule 2]
   - Flags raised: [list flags raised, or "None"]
   - Timestamp: [current date/time]
```

Two problems:
1. Field order is wrong: schema order is Command → Timestamp → Measure → Outcome. Current order puts Timestamp last and inserts non-schema fields (Rules applied, Flags raised) between Measure and Timestamp.
2. Non-schema fields (`Rules applied`, `Flags raised`) do not exist in the `.pbi-context.md` `## Last Command` schema and should be folded into the `Outcome` field.

Replace the `## Last Command` update block with:
```
2. Update the **Last Command** section with these four lines in this exact order:
   - Command: /pbi optimise
   - Timestamp: [current UTC ISO 8601]
   - Measure: [measure name]
   - Outcome: Optimised — [list of rule numbers applied, e.g. Rule 1, Rule 2]; Flags: [flags raised, or "None"]
```

The Command History row (around line 261) currently reads:
```
- Format: `| [timestamp] | /pbi optimise | [measure name] | Optimised — [rules applied] |`
```
This column order (timestamp | command | measure | outcome) is already correct per the schema. No change needed to this line.

Write the full file back with only the `## Last Command` block updated in Step 9. All other content must be preserved exactly.
