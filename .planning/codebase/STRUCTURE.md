## Directory Purposes

**.claude/**
- Purpose: Claude Code skill definition directory
- Contains: Skill manifest, command files, shared resources
- Structure: Single skill `pbi/` with router and command architecture

**.claude/skills/pbi/**
- Purpose: Root of the PowerBI DAX co-pilot skill
- Contains: Router file, 14 command subcommands, shared API notes
- Key files: `SKILL.md` (router), `commands/*.md` (subcommands)

**.claude/skills/pbi/commands/**
- Purpose: Individual command implementations
- Contains: 14 subcommand files, each a complete markdown instruction set
- Commands:
  - **Paste-in** (work anywhere): explain.md, format.md, optimise.md, comment.md, error.md, new.md
  - **PBIP file-mode**: load.md, audit.md, diff.md, commit.md, edit.md, undo.md, comment-batch.md, changelog.md

**.claude/skills/pbi/shared/**
- Purpose: Shared knowledge resources
- Contains: API reference for DAX Formatter, shared patterns
- Key files: `api-notes.md` (DAX Formatter integration guide)

**.planning/codebase/**
- Purpose: Codebase analysis documents for GSD pipeline
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: `/gsd:map-codebase` command

**.planning/phases/**
- Purpose: Phase execution plans from `/gsd:plan-phase`
- Contains: 5 planned phases with implementation details
- Examples: `01-paste-in-dax-commands/`, `02-context-detection-and-pbip-file-i-o/`

**tests/fixtures/**
- Purpose: Sample PBIP projects for testing command behavior
- Contains: 4 different project formats + context test file
- Projects:
  - `pbip-tmdl/`: TMDL format with git repo, 3 tables, relationships, bidirectional filter
  - `pbip-tmsl/`: TMSL format (model.bim)
  - `pbip-empty-model/`: Tables defined but no measures
  - `pbip-no-repo/`: TMDL format with no git repo (tests git init)
- Fixtures: `.SemanticModel/` directory structure matching real Power BI projects

## Naming Conventions

**Files:**
- Command files: lowercase with hyphens (`explain.md`, `comment-batch.md`)
- Documentation: camelCase or UPPERCASE (`README.md`, `CLAUDE.md`, `ARCHITECTURE.md`)
- Session memory: kebab-case (`.pbi-context.md`)
- Test fixtures: directory names describe format and git state (`pbip-tmdl`, `pbip-tmsl`, `pbip-no-repo`)

**Directories:**
- Skill structure: lowercase (`.claude/skills/pbi/commands/`)
- Planning artifacts: lowercase with numbers for phases (`01-paste-in-dax-commands`)
- Test fixtures: descriptive kebab-case (`pbip-tmdl`, `pbip-empty-model`)

**Subcommands:**
- Paste-in commands: single words or verbs (explain, format, optimise, comment, error, new)
- PBIP commands: action names or gerunds (load, audit, diff, commit, edit, undo, comment-batch, changelog)

## Where to Add New Code

**New Paste-in Subcommand:**
- Implementation: `.claude/skills/pbi/commands/[new-cmd].md`
- Update: Add keyword match to routing table in `.claude/skills/pbi/SKILL.md`
- Pattern: Follow existing command format (heading, detection use, tool invocations, output)

**New PBIP File-mode Subcommand:**
- Implementation: `.claude/skills/pbi/commands/[new-cmd].md`
- Update: Add keyword match to routing table in `.claude/skills/pbi/SKILL.md`
- Decide: Sonnet (direct) or Haiku (Agent spawn) based on task type
- Pattern: Include instructions for reading/writing model files, updating session context

**New Shared Resource:**
- Location: `.claude/skills/pbi/shared/[resource-name].md`
- Update: Reference in relevant commands that consume it

**Test Fixtures:**
- Location: `tests/fixtures/[scenario-name]/`
- Structure: Mirror real PBIP structure (`.SemanticModel/definition/tables/`, `.git/`, etc.)
- Purpose: Enable command testing with real Power BI file formats

**Planning Artifacts:**
- Codebase analysis: `.planning/codebase/[TOPIC].md`
- Phase plans: `.planning/phases/[NN-phase-name]/` directory
- Research: `.planning/research/[topic].md`

## Special Directories

**.claude/**
- Purpose: Claude Code skill system directory
- Generated: No (user-maintained)
- Committed: Yes (contains skill definitions)
- Note: Skill discovery is automatic; any skill in `.claude/skills/*/` is available in Claude Code

**.planning/**
- Purpose: GSD planning artifacts (codebase analysis, phase plans, research)
- Generated: Yes (by GSD commands)
- Committed: Yes (planning docs are version-controlled)
- Note: Output of `/gsd:map-codebase`, `/gsd:plan-phase`, `/gsd:execute-phase`

**tests/fixtures/**
- Purpose: Sample PBIP projects for testing command behavior
- Generated: No (manually created)
- Committed: Yes (used for regression testing)
- Note: Each fixture is a complete Power BI project directory with `.SemanticModel/`, `.git/`, optional `.Report/`

**.git/**
- Purpose: Git repository for version control and command history
- Generated: Yes (by git init)
- Committed: N/A (git metadata)
- Note: Commands write to git log; `changelog` command reads git history
